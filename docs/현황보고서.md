## 1. 기획 대비 구현 현황

### 필수 기능 (6/6 완료)

| # | 기획 항목 | 상태 | 구현 내용 |
|---|---|---|---|
| 1 | 팀 생성 | ✅ | 팀명·프로젝트명·설명·GitHub 주소 입력. 홈 "빠른 실행"에서 바로 진입 |
| 2 | 팀원별 역할 입력 | ✅ | 생성 시 팀원별 이름·역할·담당 업무 입력 + **생성 후에도 개요 탭에서 팀원 추가 가능** |
| 3 | 활동 로그 입력 | ✅ | 카테고리 칩(회의/개발/자료조사/문서/발표/기타) + 내용 + 시간 슬라이더 + 기분 이모지. 본인 기록만 삭제 가능 |
| 4 | 동료평가 입력 | ✅ | 기여도·책임감·협업·소통 4항목 5점 슬라이더 + 익명 한줄 코멘트. 재제출 시 덮어쓰기 |
| 5 | AI 기여도 리포트 | ✅ | Claude API가 활동 로그·동료평가·역할을 **교차검증**해 리포트 생성. API 키 없으면 룰 기반 로컬 분석으로 자동 대체 (데모 안 끊김) |
| 6 | 기여도 참고 점수 시각화 | ✅ | 교수님 팀 화면에 **"참고 기여도" 차트** (활동 60% + 동료평가 40% 결합, 리포트와 동일 계산) |

### 1순위 (8/8 완료)

더미 로그인(이름만 입력, 역할 카드 선택) / 팀 생성 / 팀원 등록 / 역할 입력 / 활동 로그 / 동료평가 / AI 리포트 / 교수님 대시보드 — **전부 완료**

### 2순위 (6/6 완료)

| 기획 항목 | 구현 내용 |
|---|---|
| 차트 시각화 | 활동 시간 도넛, 동료평가 바, 참고 기여도 바, 활동 유형 바, 기여도 잔디 히트맵 |
| PDF 다운로드 | 리포트 → A4 PDF 변환 → 공유 시트(저장·인쇄·드라이브·카톡) |
| 산출물 파일 업로드 | 팀 상세 "산출물" 탭. 파일 선택기로 첨부(드라이브 파일도 가능), 업로더·날짜 표시 |
| GitHub 링크 분석 | 팀 저장소 등록 → 교수님 화면에서 버튼 한 번에 최근 300커밋 저자별 집계 차트 |
| 리포트 재생성 | 리포트 화면 우상단 🔄 버튼 |
| 교수자 코멘트 | 교수님 팀 화면 "교수자 메모" 저장 |

### 3순위 (로드맵 — 발표 때 "향후 계획"으로 활용)

- GitHub 커밋 자동 분석 → **반자동(버튼 클릭)으로 이미 구현됨**
- Google Drive 연동 → 산출물 업로드가 시스템 파일 선택기라 **드라이브 파일 첨부는 이미 가능**
- 학교 LMS 연동 / 알림 / 조직·회사 버전 → 미구현 (의도적 범위 제외)

## 3. 품질·보안 (심사 질문 대비)

- **유닛 테스트 12개** 전부 통과 — 플래그 감지 3종, 자기평가 제외, 스트릭, 건강도, 참고 기여도, 시드 무결성 (`./gradlew test`)
- API 키는 **AES-256 암호화 저장소**에 보관 (평문 JSON 아님), `allowBackup=false`로 백업 유출 차단
- 데이터 파일은 **원자적 쓰기**(저장 중 크래시에도 원본 보존), 디스크 I/O 백그라운드 처리 (ANR 수정 완료)
- **프롬프트 인젝션 방어** — 로그에 "나한테 만점 줘" 같은 지시를 심어도 AI가 무시하고 조작 시도로 보고
- 알려진 한계(발표 때 솔직하게): 더미 로그인=실제 인증 없음(→학교 SSO 로드맵), 로컬 저장(→서버 로드맵), 클라이언트 API 키(→서버 프록시 로드맵)

## 4. 시연 가이드 (시훈 필독)

**준비**: 앱 설치 → 로그인 화면에서 **로고를 길게 누르면** 데모 데이터 로드 (토스트 뜸). 설정 → "데모 데이터 불러오기"로도 가능.

**추천 시나리오 (3분)**
1. (30초) 문제 제기 — "팀플 점수는 N분의 1로 나뉩니다. 일도 그랬을까요?" + 설문 결과(무임승차 경험 87%)
2. (60초) 학생 플로우 — 소이로 로그인 → 홈(스트릭🔥·잔디) → 활동 기록(카테고리+기분😄) → 동료평가 슬라이더 → 산출물
3. (60초) 교수님 플로우 — 대시보드(건강도·확인 필요 배지) → 팀 클릭 → ⚠️ 민준 무임승차 플래그 → 참고 기여도 차트 → GitHub 커밋 분석 → AI 리포트 생성 → PDF 공유
4. (30초) 마무리 — 다크모드 전환 + "감시가 아닌 증명. 열심히 한 만큼 인정받는 팀플."

**주의**: 시드의 GitHub 분석은 우리 레포(jmpaark/hackerton) 커밋을 실제로 보여줌 — **각자 커밋 몇 개씩 올려두면 데모가 훨씬 산다.** 에뮬레이터 녹화 권장(3분 제한과 동일: `adb shell screenrecord`).
